01
Overview & Objective

This SOP establishes a unified naming convention and structural logic for all tracking deployments. The goal is data consistency across every client account — so any analyst can open a GA4 report or BigQuery table and immediately understand what they're looking at, without a decoder ring.

Three core components work together in every deployment:

ComponentDefinition
EventAny user interaction with a digital product — a click, scroll, form submit, video play.
Event TaxonomyThe structured naming system that makes events searchable, filterable, and reportable in GA4 and BigQuery.
GTM Data TagThe bridge between the site's Data Layer and the destination platform. Uses Triggers to listen for actions and Variables to capture context.
02
The Four Pillars of Taxonomy Design

Every tag and event created must satisfy all four of these criteria. Failing any one of them creates "tracking debt" — inconsistencies that compound over time and corrupt reporting.

Consistent
Same casing and syntax enforced across the entire account. One deviation creates duplicate rows.
Scalable
Names must work for one button or one thousand buttons without restructuring the taxonomy.
Clear
A stakeholder reading a report must immediately understand the interaction — no dictionary required.
Intuitive
New events must fit naturally into the existing hierarchy by logic, not guesswork.
03
Taxonomy Design Rules — The Syntax
A. Casing Standard

All event names use snake_case to align with GA4's default architecture. PascalCase is only permitted if explicitly requested by developers for a technical integration.

FormatExampleWhen to Use
snake_case click_nav_footer_cta Standard — use always
PascalCase ClickNavFooterCta Only when developers require it for technical alignment
Mixed click_nav_FooterButton Never — creates duplicate rows in reports
B. Structure: Object + Action

To align with GA4 defaults, event names follow an Object first, Action second structure. This ensures events group logically when sorted alphabetically in reports.

Example
✓ Correctform_submit   video_play   file_download
✗ Incorrectsubmit_form   play_video
C. The "Actor" Rule — User Perspective

Names must always reflect the user's action, not the system's. This prevents ambiguity in reports about whether an event was triggered manually or automatically.

NameProblem / Why It Works
✗ Ambiguousmessage_sentDid the user send it, or did the system send it automatically?
✓ Clearuser_message_submitUnambiguous — this is a human action, tracked in real time.
D. Grammar Rules
RuleCorrectIncorrect
Singular nounsclick, formclicks, forms
Present tense verbsclick, submitclicked, submitted
Consistent terminologyPick one: cta, button, or btnUsing all three across the same project
04
Hierarchy & Inheritance

The taxonomy follows a Parent-to-Child logic. This allows analysts to query at any level of granularity — from total clicks site-wide all the way down to a specific footer CTA — without restructuring the data.

Formula: [Root Action] _ [Category] _ [Component] _ [Element]
▸ click Level 1 — Root Action (what type of interaction?)
▸ click_nav Level 2 — Category (where on the page?)
▸ click_nav_footer_cta Level 3 — Element (which specific element?)
Why this matters in reporting

Using click_nav as a filter returns every navigation click. Using click_nav_footer_cta isolates just that element. One taxonomy supports both a CMO-level overview and a developer-level debug in the same GA4 property.

05
E-Commerce Taxonomy (GA4 Recommended Events)

For e-commerce clients, the GA4 Recommended Event schema is mandatory for the purchase funnel. Custom event names for these interactions are not permitted — deviating from the standard breaks GA4's built-in e-commerce reports and Ads integration.

GTM Tag Naming & Master Tag Logic
Tag Naming in the GTM UI

All tags follow the format [Platform] — [Type] — [Description] to make the GTM workspace scannable at a glance.

Tag Naming Examples

Format: [Platform] - [Type] - [Description]

GA4 - Event - Navigation Clicks       // ✓ Correct format
Meta - Pixel - Purchase               // ✓ Correct format
Navigation Click Tag                  // ✗ No platform, no type
Master Tag Logic — Avoiding Tag Bloat

Rather than building 50 individual tags for 50 buttons, we use a single Master Tag with dynamic variables. This keeps the GTM container clean, reduces maintenance overhead, and makes debugging faster.

GA4 — Event — Global Click Handler

One tag handles all click events dynamically via lookup table or regex variable.

Tag Name:    GA4 - Event - Global Click Handler
Event Name:  {{Event Name Hierarchy}}   // Lookup table or regex variable
Parameter:   click_text   {{Click Text}}
Parameter:   click_url    {{Click URL}}
07
Taxonomy Creation Workflow

When starting a new client deployment, work through these six phases in order. Each phase builds on the last — skipping ahead creates gaps that are expensive to fix later.

Phase 1
Business Profile Diagnostic
  • What is the business model? (E-com, Lead Gen, SaaS)
  • What is the primary conversion?
  • What are the secondary high-intent actions?
Phase 2
Object & Interaction Mapping
  • UI navigation categories (Main Nav, Footer, Sidebar)
  • Form types (Contact, Quote, Login)
  • Interactive assets (Videos, PDFs, Carousels)
Phase 3
Hierarchy Builder
  • Step 1: Choose Root Action (click, form, view, video)
  • Step 2: Add Category (nav, body, header, footer)
  • Step 3: Add Component (cta, link, social, lead)
  • Step 4: Add Element if needed (unique identifier)
Phase 4
Model-Specific Application
  • E-com: Use GA4 standard names as root (view_item, add_to_cart). Custom: click_body_product_card
  • Lead Gen: form_lead_start, form_lead_submit, click_contact_phone
Phase 5
Parameter Assignment
  • click_nav_footer_social → social_platform, link_url
  • form_lead_submit → form_id, form_location
Phase 6
Final Review Checklist
  • Everything in snake_case?
  • Object + Action structure?
  • Root is a standard GTM Trigger type?
  • Readable without seeing the website?

Overview & Objective

UTM codes are the foundation of traffic attribution. Without them, GA4 collapses paid campaigns, email flows, and influencer traffic into a single "Unassigned" bucket — making it impossible to measure channel ROI or optimize spend allocation.

This SOP ensures every external link is tagged consistently, every parameter follows GA4 conventions, and no session data is accidentally overwritten by internal tagging errors.

https://website.com/page/?utm_source=facebook&utm_medium=paid_social&utm_campaign=summer_sale&utm_content=hero_banner

02
The 5 Core Parameters
ParameterDescriptionExample Values
utm_sourceThe specific entity sending traffic — WHOgoogle, facebook, klaviyo, qr_flyer
utm_mediumThe high-level channel — HOW it arrivedcpc, email, paid_social, print, sms
utm_campaignThe specific marketing effort or promotionsummer_sale_2024, abandoned_cart
utm_termPaid keywords (Search) or audience segmentrunning_shoes, prospecting_lookalike
utm_contentDifferentiates ads or link locations within a campaignblue_banner, link_footer, video_ad

The Golden Rule — Source vs. Medium: Mixing these up is the #1 cause of dirty attribution data. utm_source = WHO sent the traffic (e.g., klaviyo). utm_medium = HOW it arrived (e.g., email). If you reverse them, GA4 will not recognize the channel and the session disappears into "Unassigned."

03
Strategic Best Practices
A. Never Tag Internal Links

UTM tags must never be used on links that point from one page of the client's site to another. If a user arrives via Google Search and then clicks a UTM-tagged internal banner, the original organic session is killed — and the purchase gets attributed to the banner instead of the search campaign.

Solution: Use GTM Event Tracking (see the Event Taxonomy SOP) to track internal click interactions. UTMs are for external traffic entry points only.

B. Casing & Syntax Rules
RuleCorrectIncorrect
Lowercase onlyemailEmail, EMAIL — GA4 treats these as 3 separate channels
No spacessummer_salesummer sale — becomes summer%20sale in the URL
No abbreviationsfacebookfb — breaks consistency as the team grows
C. Platform-Specific Rules
PlatformRule
Google Ads If Auto-tagging (GCLID) is enabled and GA4/Google Ads are linked, do not manually tag. Exception: only use manual UTMs to pass custom data to a non-Google tool like a CRM.
Meta (Facebook/Instagram) Be cautious with dynamic tags like {{campaign.name}} — these have been observed failing and appearing literally in GA4. For high-stakes campaigns, use manual tagging in the URL Suffix field instead.
Print / Offline Use QR codes. Always attach UTMs first, then shorten the link. Use utm_medium=print or utm_medium=qr to distinguish from digital traffic.
